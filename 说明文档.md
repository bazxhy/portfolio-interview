# 个人介绍网站 - 项目说明

## 1. 使用了哪些AI工具

本项目使用 **Claude Code**（Anthropic 的 AI 编程助手，通过 OpenCode 客户端）完成开发，具体包括：
- **Claude Code**：负责整体代码生成、样式设计、交互逻辑编写、问题排查与修复、GitHub Pages 部署
- **OCR 文字识别**：用于从题目图片中提取文字需求

## 2. AI帮助完成了哪些工作

AI 帮助完成了以下核心工作：
- ✅ **需求分析**：从图片中提取题目要求，拆解为清晰的任务列表
- ✅ **项目结构设计**：规划了单页网站（入场动画、导航、首屏、关于、技能、项目、联系、页脚）
- ✅ **HTML 代码生成**：生成了语义化、结构清晰的 HTML5 代码，包含真实个人信息与中英双语支持
- ✅ **CSS 样式设计**：设计了深色/浅色双主题，玻璃态（glassmorphism）卡片、渐变、动画进度条等现代效果
- ✅ **JavaScript 交互开发**：实现了水墨入场动画（Canvas 粒子 + 打字机字幕）、四季动态生态系统（春/夏/秋/冬随滚动切换）、滚动触发动画、数字计数动画、技能条动画、导航高亮、移动端菜单、中英文切换等
- ✅ **UI 布局优化**：将中英文切换提取为独立组件置于页面右上角，方便用户快速操作
- ✅ **问题排查与修复**：诊断并修复线上部署失效问题（Pages 工作流失败 + gh-pages 分支不同步）
- ✅ **部署上线**：通过 GitHub Pages（gh-pages 经典分支部署）完成永久部署

## 4. 遇到的问题，以及如何通过AI解决

| 问题 | AI 解决方案 |
|------|------------|
| 题目是图片格式，无法直接读取文字 | 使用 OCR 文字识别提取中文题目要求 |
| 需要快速构建美观的个人网站 | AI 生成了深色玻璃态设计，参考现代 portfolio 风格，包含丰富的 Canvas 动画 |
| Netlify Drop 需要每次手动拖拽 | 改用 GitHub Pages + gh-pages 分支实现永久自动部署 |
| 页面内容单调，需要更丰富 | AI 添加了水墨入场动画、打字机效果、四季粒子生态系统、计数动画、技能进度条等 |
| 线上四季动态效果全部丢失 | 发现 GitHub Pages 从 gh-pages 分支部署，而该分支仍是旧版本（缺少 seasons.js、ink-intro.js），通过 `git push origin master:gh-pages --force` 同步修复 |
| 欢迎入场动画不显示 | 同上，ink-intro.js 文件在旧 gh-pages 分支不存在，同步后恢复水墨粒子入场效果 |
| 中英文切换隐藏在设置面板内，不直观 | AI 新增独立语言切换栏置于页面右上角，同时修复 i18n 状态类名（`.active` → `.on`）同步问题 |
| 修改后网站出现 404（GitHub Pages 未配置） | AI 排查发现 GitHub Actions 工作流（pages.yml）持续运行失败导致 Pages 部署阻塞，移除失败工作流后经典 gh-pages 分支部署自动恢复 |
| GitHub 仓库默认分支（main）为空，找不到代码 | AI 将 master 分支同步到 main 分支，确保在 GitHub 网页端默认显示完整代码 |
| 切换英文后姓名仍显示中文 | AI 为全站姓名（首页大字、关于区域、页脚）添加 `data-i18n` 翻译标记，同时注册中英文翻译键值（关福博 / Guan Fubo） |
| 设置面板内中英文切换与右上角语言栏重复 | AI 移除设置面板中的重复语言切换区域，仅保留主题切换，实现语言栏与设置面板职责分离 |
