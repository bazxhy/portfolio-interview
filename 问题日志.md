# 问题排查日志

## 问题1：线上四季动态效果全部丢失

**时间**：2026-06-05

**现象**：网站能打开，但欢迎入场动画（水墨效果）和四季背景动效（花瓣、蝴蝶、草地等）完全不显示。

**排查过程**：
1. 抓取线上 HTML 发现脚本引用为 `fluid-bg.js`（旧版 WebGL），非 `ink-intro.js` + `seasons.js`（新版 Canvas 动画）
2. 检查 GitHub 仓库，发现 `master` 分支有最新代码，但 `gh-pages` 分支仍是旧版本
3. 根因：GitHub Pages 从 `gh-pages` 分支部署，该分支缺少 `seasons.js` 和 `ink-intro.js`

**解决方案**：`git push origin master:gh-pages --force`

---

## 问题2：GitHub 仓库默认分支为空

**时间**：2026-06-05

**现象**：用户打开 https://github.com/bazxhy/portfolio-interview 只看到一个 README.md，找不到任何代码文件。

**排查过程**：
1. GitHub 仓库默认分支是 `main`，该分支仅有一份 README
2. 所有代码都在 `master` 分支上

**解决方案**：`git push origin master:main --force`

---

## 问题3：网站出现 404（GitHub Pages 未配置）

**时间**：2026-06-05

**现象**：修改中英文切换位置后，网站突然返回 GitHub 404 页面："There isn't a GitHub Pages site here."

**排查过程**：
1. 检查 GitHub REST API `/repos/bazxhy/portfolio-interview/pages`，返回 404（Pages 未配置）
2. 检查 GitHub Actions 工作流运行状态，发现 14 次运行全部 `"conclusion": "failure"`
3. 根因：`.github/workflows/pages.yml` 持续失败，阻塞了 Pages 部署

**解决方案**：删除 `.github/workflows/pages.yml`，改用经典 `gh-pages` 分支部署，Pages 自动恢复

---

## 问题4：中英文切换状态不同步

**时间**：2026-06-05

**现象**：页面初始化时语言显示正确，但点击切换后按钮高亮状态错乱（中文按钮高亮但实际已是英文）。

**排查过程**：
1. 检查 `i18n.js` 的 `updateButtons()` 方法，发现使用 `classList.toggle('active', ...)`
2. 检查 CSS，发现活跃状态样式用的是 `.lang-btn.on`，而非 `.lang-btn.active`
3. 根因：JS 与 CSS 使用的类名不一致（`.active` vs `.on`）

**解决方案**：将 `i18n.js` 中 `updateButtons()` 的类名从 `active` 改为 `on`

---

## 问题5：切换英文后姓名仍显示中文

**时间**：2026-06-05

**现象**：点击右上角 EN 切换到英文后，首页大字"关福博"、关于区域姓名、页脚姓名仍显示中文。

**排查过程**：
1. 检查 HTML，发现姓名是硬编码文本，未使用 `data-i18n` 属性
2. 根因：姓名字段未被纳入国际化翻译系统

**解决方案**：
- 为首字姓名（关/福/博）添加 `data-i18n` 属性并注册翻译键值
- 为关于区域和页脚姓名添加 `data-i18n` 标记
- 中文值：关福博，英文值：Guan Fubo

---

## 问题6：语言切换栏与设置面板功能重复

**时间**：2026-06-05

**现象**：右上角已有独立语言切换栏（中文/EN），但右下角设置面板内还有一套语言切换按钮，功能重复且界面冗余。

**解决方案**：从设置面板（`#stgPanel`）中移除语言切换区域，仅保留主题切换（深色/浅色）

| 序号 | 问题 | 根因 | 解决方式 |

| 1 | 四季动效丢失 | gh-pages 分支代码陈旧 | force push master → gh-pages |

| 4 | 语言切换状态错乱 | JS 类名 `.active` ≠ CSS `.on` | 统一为 `.on` |
